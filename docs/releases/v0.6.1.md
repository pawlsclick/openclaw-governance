# openclaw-governance v0.6.1

Patch release addressing [Issue #21 validation feedback](https://github.com/pawlsclick/openclaw-governance/issues/21#issuecomment-4583049749).

## Fixes

### Skill JSON capture (pipe truncation)

`openclaw skills list --json` can exceed pipe buffer limits when captured with `subprocess.run(capture_output=True)`. Redirecting to a file works; v0.6.1 switches `run_openclaw_json` to temp-file stdout capture for all JSON CLI calls.

### Artifact consistency on degraded discovery

When CLI capture fails, v0.6.0 still ran filesystem skill scan and printed a summary, but refused to write `workflows/discovered-skills.json` because discovery errors were treated as blocking. v0.6.1 writes the artifact with:

- `cli_capture`: `"ok"` or `"failed"`
- `degraded`: `true` when CLI phase failed but FS fallback produced skills
- `errors`: structured failure records (unchanged)

`check --skills` can then run against the committed artifact instead of failing on a missing file.

## Unchanged

- Plugin discovery and `check --plugins` behavior (validated working on operator host).
- Read-only discover without `--inventory` / `--staged`.
- Registry auto-promotion policy (still manual review).

## Upgrade

```bash
pipx install "openclaw-governance @ git+https://github.com/pawlsclick/openclaw-governance@v0.6.1" --force
```

Re-run staged discovery:

```bash
openclaw-gov discover --staged --include-skills --include-plugins
```

Verify skills artifact:

```bash
jq '{cli_capture, degraded, total: .summary.total}' workflows/discovered-skills.json
```

## Operator follow-up (config, not package)

Add `capabilities.expected_plugins` for enabled plugins flagged undocumented during validation, for example:

```yaml
capabilities:
  expected_plugins:
    - active-memory
    - google
    - mattermost
    - memory-core
    - nvidia
    - openai
    - telegram
    - venice
    - acpx
    - amazon-bedrock
    - brave
    - codex
    - lossless-claw
  check_fail_on:
    - undocumented_plugin_enabled
```

Tune `expected_skills` after CLI capture succeeds and eligibility metadata is present.

# openclaw-governance v0.7.4

Patch release for [Issue #33 — plugin-scoped agents forced into platform RACI](https://github.com/pawlsclick/openclaw-governance/issues/33).

## Fix

Plugin-owned agents (e.g. mnemospark) no longer default into global platform RACI broadcast lists. `openclaw-gov check` will not require them in `platform_notion.informed` or `platform_google.informed` unless you explicitly promote them.

Detection uses `openclaw plugins list --json`:

- agent id matches a plugin id, or
- agent workspace path is under a plugin `rootDir`

Registry fields set on discover/promote:

```yaml
governance_scope: plugin
raci_broadcast_excluded: true
```

To promote a plugin agent to global broadcasts:

```yaml
governance_scope: core
raci_broadcast_excluded: false
```

Or add the agent id to `agents.broadcast_excluded: []` removal path via explicit promotion above.

## Upgrade

```bash
pipx install "openclaw-governance @ git+https://github.com/pawlsclick/openclaw-governance@v0.7.4" --force
openclaw-gov discover --promote   # refreshes plugin scope on existing rows
openclaw-gov check
```

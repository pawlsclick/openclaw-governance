# openclaw-governance v0.7.2

Patch release for [Issue #30 — promote leaves discovered_from.cron_fingerprint stale](https://github.com/pawlsclick/openclaw-governance/issues/30).

## Fix

When `discover --promote` updates a cron-backed workflow's fingerprint, all three registry locations now stay aligned:

- `workflow.cron_fingerprint`
- `workflow.discovered_from.cron_fingerprint`
- `workflow.discovered_from.cron_instances[].fingerprint`

`openclaw-gov check` also flags workflows where the top-level and nested cron fingerprints disagree (single-string rows).

## Upgrade

```bash
pipx install "openclaw-governance @ git+https://github.com/pawlsclick/openclaw-governance@v0.7.2" --force
```

After upgrade, re-run `discover --promote` on affected workflows or manually align any stale nested fingerprints caught by `check`.

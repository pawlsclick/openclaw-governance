# Release notes: openclaw-governance v0.5.5

Target audience: operators maintaining a brownfield governance repo (for example `openclaw-workspace-governance`).

## Install

```bash
pipx install "openclaw-governance @ git+https://github.com/pawlsclick/openclaw-governance@v0.5.5" --force
openclaw-gov --version   # 0.5.5
```

## What changed

### Read-only discover by default

| Command | Writes |
|---------|--------|
| `discover` | Nothing (console only) |
| `discover --inventory` | `workflows/discovered-inventory.json` |
| `discover --staged` | inventory + `workflows/discovery-candidates.json` |
| `discover --promote` / `--write` | inventory (+ registry/runbooks per existing rules) |

Optional: `--include-runtime-metrics` adds `workflows/discovered-inventory-runtime.json` (gitignore locally; not for committed inventory).

### Stable inventory (schema v2)

Committed inventory no longer includes:

- `generated_at`
- `agent_statuses` / `duration_ms`

Those live in the optional runtime file when requested.

Inventory shape changes:

- `inventory_schema_version: 2`
- `group_id` on jobs and `cron_instance_groups` (replaces `group_key` / `instance_group_key`)
- `schedule` as JSON object when OpenClaw returns structured schedules
- `message_preview` redacts sensitive flag values (`--wallet-address`, `--token`, etc.)

### Config extension

```yaml
discovery:
  sensitive_preview_flags:
    - "--my-custom-secret-flag"
```

## Brownfield upgrade checklist

1. Upgrade the CLI to v0.5.5.
2. Run `openclaw-gov discover` and confirm the working tree stays clean (read-only).
3. Run `openclaw-gov discover --staged --root .` (or `--inventory` if you only need inventory).
4. Review the diff to `workflows/discovered-inventory.json`:
   - Expect schema changes even when cron jobs are unchanged.
   - Confirm wallet/token-like values show `<redacted>` in previews.
5. Commit the regenerated inventory when satisfied.
6. Update downstream CI if it pinned an older tag (template now uses `@v0.5.5`).

Promote behavior from v0.5.3/v0.5.4 (preserve curated agents/RACI, skip existing runbook stubs) is unchanged.

## jq examples (schema v2)

Fan-out groups:

```bash
openclaw-gov discover --json | jq '.cron_instance_groups[] | select(.kind == "fan_out")'
```

Match a job to its group:

```bash
jq '.cron_instance_groups[] | select(.group_id == "abc123")' workflows/discovered-inventory.json
```

## Breaking changes for inventory consumers

If you parse `discovered-inventory.json` outside `openclaw-gov`:

- Drop reliance on `generated_at` in inventory (use runtime file or git commit time).
- Replace `group_key` / `instance_group_key` with `group_id`.
- Parse `schedule` as object or string (not JSON-in-JSON string for dict schedules).

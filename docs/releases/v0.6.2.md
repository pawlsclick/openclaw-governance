# openclaw-governance v0.6.2

Patch release addressing [Issue #21 — v0.6.1 validation](https://github.com/pawlsclick/openclaw-governance/issues/21#issuecomment-4583110493).

## Fix

Runtime CLI skill records often have an empty `install_path` when OpenClaw does not expose `filePath` or `baseDir`. Duplicate detection called `Path("").resolve()`, which resolves to the **current working directory**. Every pathless skill shared that key, so the first skill (e.g. `1password`) became the canonical entry and 100+ unrelated skills were marked `duplicate_of: "1password"`.

v0.6.2 skips duplicate marking when `install_path` is blank. Real filesystem duplicates (same resolved install path) are unchanged.

## Upgrade

```bash
pipx install "openclaw-governance @ git+https://github.com/pawlsclick/openclaw-governance@v0.6.2" --force
openclaw-gov discover --staged --include-skills --include-plugins
jq '.summary' workflows/discovered-skills.json
```

Expect duplicate count near zero unless you have genuine duplicate installs on disk.

## Operator follow-up (unchanged)

Add `capabilities.expected_plugins` for enabled plugins flagged by `check --plugins`. See [v0.6.1 release notes](v0.6.1.md).

# openclaw-governance v0.6.3

Patch release addressing [Issue #21 — v0.6.2 validation](https://github.com/pawlsclick/openclaw-governance/issues/21#issuecomment-4583132269).

## Fix

Workspace skills were appearing twice in capability inventory:

1. Runtime CLI record (`source: openclaw-workspace`, often empty `install_path`)
2. Filesystem scan record (`source: workspace-scan`, concrete `install_path`)

That inflated totals (e.g. 153 vs ~107 runtime-visible skills) and duplicated `check --skills` warnings for the same skill name.

v0.6.3 merges `workspace-scan` enrichment into matching `openclaw-workspace` runtime records by name. Separate filesystem rows are kept only when the skill name is absent from runtime inventory (true orphans).

## Upgrade

```bash
pipx install "openclaw-governance @ git+https://github.com/pawlsclick/openclaw-governance@v0.6.3" --force
openclaw-gov discover --staged --include-skills --include-plugins
jq '.summary.total' workflows/discovered-skills.json
```

Expect skill totals near runtime eligible count, not roughly double workspace skills.

## Operator follow-up (unchanged)

Add `capabilities.expected_plugins` for enabled plugins flagged by `check --plugins`. See [v0.6.1 release notes](v0.6.1.md).

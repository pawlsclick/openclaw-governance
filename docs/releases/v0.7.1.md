# openclaw-governance v0.7.1

Patch release for [Issue #28 — promote rewrites discovered_at on every run](https://github.com/pawlsclick/openclaw-governance/issues/28).

## Fix

Re-running `discover --promote --include-skills --include-plugins` with an unchanged active surface no longer rewrites every capability row's `discovered_at` timestamp.

- `discovered_at` is set when a capability registry object is **first created**
- Existing rows keep their original `discovered_at` on subsequent promotes
- Material metadata (`install_path`, `source`, `enabled`, etc.) still refreshes when it changes
- Volatile run timestamps remain in inventory artifacts (`discovered-skills.json`, `discovery-candidates.json`)

## Upgrade

```bash
pipx install "openclaw-governance @ git+https://github.com/pawlsclick/openclaw-governance@v0.7.1" --force
```

After upgrade, a second promote with no capability changes should produce an empty `git diff` on `workflows/registry.yaml` (aside from unrelated workflow metadata).

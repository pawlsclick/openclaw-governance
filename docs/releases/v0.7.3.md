# openclaw-governance v0.7.3

Patch release fixing CLI version display after v0.7.2.

## Fix

`pipx install ...@v0.7.2` reported package version **0.7.2** but `openclaw-gov --version` still printed **0.7.1** because `__init__.py` hardcoded the old string.

v0.7.3 reads `__version__` from installed package metadata (single source of truth in `pyproject.toml`).

## Upgrade

```bash
pipx install "openclaw-governance @ git+https://github.com/pawlsclick/openclaw-governance@v0.7.3" --force
openclaw-gov --version   # expect 0.7.3
```

v0.7.2 includes the cron fingerprint sync fix (#30); use v0.7.3 or newer for correct `--version` output.

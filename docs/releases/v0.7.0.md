# openclaw-governance v0.7.0

Minor release for [Issue #26 — Write active skills and plugins to governance registry](https://github.com/pawlsclick/openclaw-governance/issues/26).

## What's new

`discover --promote --include-skills --include-plugins` now writes compact **`capabilities`** entries to `registry.yaml` for the **active surface only**:

- **Skills:** `eligible: true` (runtime-usable)
- **Plugins:** `enabled: true`

Inactive, bundled, or disabled capabilities stay in JSON inventory artifacts only (`discovered-skills.json`, `discovered-plugins.json`). They are not promoted into the registry unless you manually curate them.

`discover --staged --include-skills --include-plugins` adds a **`capabilities`** section to `discovery-candidates.json` with:

- `active_skill_candidates` / `active_plugin_candidates`
- `inventory_only` counts
- `drift` candidates (filesystem-only, missing registry row, undocumented active)

`check --skills --plugins` validates registry coverage against the active surface:

- Warns on eligible undocumented skills (configurable fail)
- Fails on enabled undocumented plugins when policy requires it
- Summarizes inventory-only capabilities without requiring registry rows
- Honors `capabilities.expected_*` / `exempt_*` in `governance.config.yaml`

Promote does **not** auto-update `expected_*` lists in config and does **not** create runbook stubs for every skill/plugin.

## Registry shape

```yaml
capabilities:
  schema_version: 1
  skills:
    - id: skill:demo-skill
      name: demo-skill
      type: skill
      source: openclaw-workspace
      install_path: /path/to/skill
      eligible: true
      governance_status: undocumented
      last_discovered_at: "2026-05-30T12:00:00Z"
  plugins:
    - id: plugin:discord
      plugin_id: discord
      name: Discord
      type: plugin
      enabled: true
      governance_status: undocumented
      last_discovered_at: "2026-05-30T12:00:00Z"
```

Protected fields (`runbook`, curated `governance_status`, etc.) are preserved on merge.

## Upgrade

```bash
pipx install "openclaw-governance @ git+https://github.com/pawlsclick/openclaw-governance@v0.7.0" --force
openclaw-gov discover --staged --include-skills --include-plugins
openclaw-gov discover --promote --include-skills --include-plugins
openclaw-gov check --skills --plugins
```

Expect ~13 enabled plugins and eligible skills only in `registry.yaml` capabilities — not all discovered inventory rows.

## Other fixes

- `inventory skills|plugins` no longer raises `BrokenPipeError` when piped to a consumer that closes early (e.g. `head`).

## Operator follow-up

After promote, document high-risk capabilities by setting `runbook` or `governance_status: expected` on registry rows, or add `capabilities.expected_plugins` / `expected_skills` in `governance.config.yaml` for config-level allowlists.

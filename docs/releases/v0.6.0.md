# openclaw-governance v0.6.0

Release date: 2026-05-30

## Summary

Adds read-only **plugin and skill capability inventory** to close the governance blind spot where installed skills/plugins can drift without appearing in discovery, check, or staged review artifacts.

## New commands and flags

| Command | Purpose |
|---------|---------|
| `discover --include-skills` | Scan skills (CLI + workspace filesystem) |
| `discover --include-plugins` | Scan plugins via `openclaw plugins list --json` |
| `discover --staged --include-skills --include-plugins` | Write capability artifacts + existing staged workflow inventory |
| `check --skills` | Warn on material undocumented skills (configurable) |
| `check --plugins` | Fail on enabled undocumented plugins by default |
| `inventory skills --json` | Print `workflows/discovered-skills.json` |
| `inventory plugins --json` | Print `workflows/discovered-plugins.json` |
| `regen --include-capabilities` | README summary from committed artifacts (optional markers) |

## Artifacts

- `workflows/discovered-skills.json` — stable skill inventory (`capabilities_schema_version: 1`)
- `workflows/discovered-plugins.json` — stable plugin inventory (projected fields only; no diagnostics dump)

## Configuration

Add to `governance.config.yaml`:

```yaml
capabilities:
  expected_skills: []
  expected_plugins: []
  exempt_skills: []
  exempt_plugins: []
  optional_scan_roots: []
  check_fail_on:
    - undocumented_plugin_enabled
```

## Upgrade

1. Install `@v0.6.0`.
2. Run `openclaw-gov discover --staged --include-skills --include-plugins --root <gov>`.
3. Review and commit `discovered-skills.json` / `discovered-plugins.json`.
4. Populate `capabilities.expected_*` / `exempt_*` as needed.
5. Enable `check --skills --plugins` in your ship/CI workflow when ready.
